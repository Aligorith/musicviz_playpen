== High Priority Defect Fixes ==
* Resuming playback following clicking "stop" doesn't work?

* Fix bad resizing behaviour (i.e. requires refactoring the drawing to draw to a cache buffer, instead of directly to a frame buffer that can get corrupted) - see "v51_prompt"

* Fix scrubbing crackling issues (i.e. need gain-fading on the pipeline) - see "v53_prompt"

* Fix bug where processing pauses if/when the tab loses focus, and then completely dies when we come back and look at it (or start interacting with it)
  * Option 1:  Create a "progress popup panel, with progressbar", + button to resume / cancel stalled operations. This will work by storing the "in-progress" Promise, and triggering it to get run again if it got stalled out for whatever reason?
  (NOTE: It seems we really have to move to the web-workers architecture, which requires multiple files)

* On some machines, the cursor updates aren't reliable, e.g.:
  * Wait cursor doesn't show when doing tuning tweaks
  * Wait cursor doesn't clear after analysis, until moving the mouse

* Fix the "stutter" bug when recording sometimes, where it freezes for the first few seconds, then randomly jump starts. (Happens after adjusting brightness or something)

* When starting the script on the page, need to make sure the internal state matches what the widgets are defining (some of which may be carried over from the previous state of the page)

* Trying to click / scrub while analysing will yield an error about unattached buffer during promise
