Response
========

Apply these three fixes to the current version:

---

# 1. Fix progressive spectrogram reveal

The current version still rescales all existing columns every frame because this line is wrong:

```js
const xScale =
    canvas.width /
    cachedSpectrogram.length;
```

That causes compression as more FFT columns arrive.

Instead, compute a FIXED column width once at analysis start.

---

## ADD global:

```js
let fixedColumnWidth = 1;
```

---

## Inside `process(buffer)` AFTER:

```js
const total =
    Math.floor(
        (
            data.length -
            fftSize
        ) / hop
    );
```

ADD:

```js
fixedColumnWidth =
    canvas.width / total;
```

---

## Then REPLACE inside `redrawSpectrogram()`:

REMOVE:

```js
const xScale =
    canvas.width /
    cachedSpectrogram.length;
```

---

And REPLACE:

```js
i * xScale
```

WITH:

```js
i * fixedColumnWidth
```

---

Replace:

```js
Math.ceil(xScale)+1
```

WITH:

```js
Math.ceil(fixedColumnWidth)+1
```

---

This now:

* reveals left→right correctly
* keeps previous columns fixed in position
* prevents shrinking/compression
* behaves like proper showcqt rendering

---

# 2. Restore elapsed + remaining time estimates

Inside `process(buffer)` ADD near the start:

```js
const processingStart =
    performance.now();
```

---

Then REPLACE:

```js
statusEl.textContent =
    `Processing ${
        i
    } / ${
        total
    }`;
```

WITH:

```js
const elapsed =
    (
        performance.now() -
        processingStart
    ) / 1000;

const progress =
    (i + 1) / total;

const estimate =
    elapsed / progress;

const remaining =
    Math.max(
        0,
        estimate - elapsed
    );

statusEl.textContent =
    `Processing ${
        i + 1
    }/${
        total
    }  •  elapsed ${
        elapsed.toFixed(1)
    }s  •  remaining ≈ ${
        remaining.toFixed(1)
    }s`;
```

---
